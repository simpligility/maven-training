Building the Project

Let's build this project...

$ mvn clean install

What Just Happened?

  • Maven inspected a file called the Project Object Model (POM)
  • It downloaded dependencies.
  • It compiled Java code into bytecode.
  • It ran a series of Unit Tests.
  • It assembled everything into a JAR file.

Running the Project

target/sample-project-1.0-SNAPSHOT.jar

$ java -jar target/sample-project-1.0-SNAPSHOT.jar

A Common Structure

Source and Resources

  • Source Code:: src/main/java
  • Resources:: src/main/resources

Both directories added to classpath and resulting artifact.

Web Application

  • Web Application:: src/main/webapp

Stored in the resulting artifact.

Unit Tests

  • Unit Tests:: src/test/java
  • Test Resources:: src/test/resources

Not stored in the resulting artifact.

Coordinates

Everything has a Coordinate

  • Projects
  • Dependencies
  • Maven Plugins

Give me the Coordinates?

  1. Group ID
  2. Artifact ID
  3. Version

Group ID

Identifier: groupId

Role: Artifacts (JARs, WARs, EARs) are organized into groups.

Groups correspond to organizations and projects.

Artifact ID

Identifier: artifactId

Role: Projects produce artifacts. This project produces a WAR artifact.

A Maven project produces a single artifact.

Version

Identifier: version

Role: Software changes over time and releases need to be versioned.
